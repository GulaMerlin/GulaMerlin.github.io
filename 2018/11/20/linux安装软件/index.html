Apache
Nginx - - -> HTTP服务器 - - -> 提供Web应用

internet是基于TCPP/IP模型的四层架构

应用层 - HTTP(80) / FTP / SSH / QQ / SMTP(25) / POP3(110)
传输层 - TCP / UDP - 端到端的传输服务 （TCP可靠通信、流量控制、拥塞控制）
网络层 - IP -
物理链路层 (网络接口层) - 分帧和校验

服务端口是对IP地址的扩展，用来区分不同的服务

安装nginx

yum - y install nginx
yum -y remove nginx —删除名为nginx软件包
yum search nginx —搜索名为nginx软件包
yun update nginx —更新名为nginx软件包(不跟名字更新所有已安装的软件包)
yum info nginx —查看名为nginx软件包信息
yum list installed —查看已经安装的软件包

yun -y install httpd —安装Apache

Git

1.创建新的空仓库

git init

2.将代码纳入暂存区
git add . / 文件名

3.查看暂存区状态
git status
git rm –cached 文件名(从暂存区移除)
git checked –文件名(将暂存区文件恢复到工作区)

4.将暂存区内容提交到本地仓库
git commit -m ‘’

5.查看提交日志

git log

git reflog - 看到未来的版本(曾经提交过又被撤回的版本)
6.回到历史版本
git reset 版本代号
git reset –hard HEAD^
get reset HEAD^
git reset 版本的哈希码

—- hard参数为了保持工作区和历史版本的一致性

如果Git联网操作报错通常是因为底层依赖库libcurl安装不正确
解决这个问题的方法如下:

  1. yum -y install curl libcurl libcurl-devel
  2. cd ~
  3. cd git-2.19.1
  4. make clean
  5. ./configure –prefix=/usr/local
  6. make && make install
