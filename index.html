模型

创建模型

一对多

外键ForeignKey定义在’多’的一方

db.relationship(‘Student’, backref=’g’) 定义在’一’的那方

‘Student’ ：’一’那方的模型类名 backref反向引用的关联关系

‘一’查’多’: ‘一’的对象.relationship字段

‘多’查’一’: ‘多’的对象.backref字段

多对一

中间表s_c

relationship字段定义在任何一方都可

secondary参数指定关联的中间表

backref反向引用的关联关系

relationship字段所在的模型对象.relationshop字段，查询的结果为另外一个模型的列表

relationship字段不在的模型.backref，查询结果另一个模型刘表

lazy定义SQLAlchemy何时从数据库加载数据

默认lazy为True

分页

pg = 模型.query.paginate(那一页, 每一页数据条数)

​ 获取数据:pg.items

​ 上一页判断: pg.has_prev

​ 上一页页码: pg.prev_num

​ 下一页判断: pg.has_next

​ 下一页页码: pg.next_num

​ 总共页数: pg.pages

​ 当前页码: pg.page

​ 共多少数据： pg.total

​ 页码列表: pg.iter_pages()

钩子函数

请求之前: @bp.before_request

不要写return，如果写return将直接响应内容给浏览器

多个before_request装饰的函数将依次调用

请求之后: @bp.after_request

一定要写return reponse

程序正常执行下才会调用

多个after_requset装饰的函数将倒叙一次执行

请求之后: @bp.teardown_request

不管异常是否处理，无论如何都会执行

程序不管是否出错，都会执行

异常

引入: from flask import abort

抛出异常: abort(500)

捕获异常: @app.errorhandlers(500)

一些常用的 HTML DOM 方法：

getElementById(id) - 获取带有指定 id 的节点（元素）
appendChild(node) - 插入新的子节点（元素）
首先创建一个节点，
然后创建一个文本节点，
然后将文本节点添加到LI节点上。
最后将节点添加到列表中。
appendChild() 方法，将新元素作为父元素的最后一个子元素进行添加。
如果不希望如此，您可以使用 insertBefore() 方法

removeChild(node) - 删除子节点（元素）
